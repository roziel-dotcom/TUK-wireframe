Page strategy

Make it a waitlist lander first, not a full ecommerce PDP.

The page should validate demand, explain the dad-specific angle, capture emails repeatedly, and create reusable Kickstarter assets.

Section 01

Dad carry should feel natural, not complicated.

Open with dad-specific pain points so the positioning feels intentional right away.

1

Not built for dad bodies

Many carriers do not feel designed for broader shoulders, longer torsos, or fast fit adjustments.

2

Too much setup

Dads need a simple carrier experience, not a confusing strap puzzle during busy moments.

3

Hands are never free

Errands, airport days, weekend plans, and coffee runs are easier when carrying feels secure.

Reference logic

Inspired by Tactical Baby Gear + Mission Critical: make dads feel like this product was intentionally built for them.

Section 04

Put it on. Buckle in. Go dad mode.

Use a simple 3-step demo flow inspired by Lalabu and SNUG.

Put on the carrier

Secure baby comfortably

Show safe and supported positioning through a simple visual sequence.

Step 3 visual

Move hands-free

Show dad using both hands while baby is secure.

Section 06

Trusted for baby. Built for dad.

Balance the dad positioning with safety and quality proof so it still feels credible for a baby product.

Safety testing

Use confirmed testing claims only once approved.

Factory/PVT progress

Show where the product is in development and launch readiness.

Comfort validation

Add dad/baby fit testing or user feedback if available.

Kickstarter readiness

Explain why waitlist growth comes before campaign launch.

Important note

Avoid overclaiming safety/ergonomics until approved testing and compliance language is confirmed.
